# PRD — Festa Radar

## 1. 문제 정의

뮤직/락 페스티벌 정보가 NOL티켓(인터파크), 멜론티켓, 티켓링크, YES24 등 예매처별로
흩어져 있어, 어떤 페스티벌이 언제 열리는지 파악하려면 사이트마다 들어가 검색해야 한다.
**한 페이지에서 전국 뮤직 페스티벌 일정을 훑고, 클릭 한 번에 예매 페이지로 이동**하게 한다.

## 2. 사용자와 핵심 시나리오

- 사용자: 페스티벌 관람을 좋아하는 국내 음악 팬 (1차 사용자는 운영자 본인)
- 시나리오 A: "올여름 갈 만한 페스티벌 뭐 있지?" → 캘린더 뷰에서 7~8월 훑기
- 시나리오 B: "부산록페 예매 어디서 하지?" → 카드에서 [YES24 예매] 버튼 클릭 → 예매처 상품 페이지
- 시나리오 C: "이번에 새로 뜬 페스티벌 있나?" → 최근 등록순 정렬로 신규 항목 확인

## 3. MVP 범위 (Phase 1)

### 포함

| # | 기능 | 상세 |
|---|---|---|
| F1 | 페스티벌 목록 자동 수집 | KOPIS 축제목록 API(대중음악) + 공연목록 API 키워드 필터 병합. 조회 범위: 오늘 −30일 ~ +365일 |
| F2 | 예매처 직링크 | 상세 API `relates` 필드. 예매처명 정규화(예스24→YES24 등) 후 버튼으로 표시. relates 비어있으면 예매처 검색 딥링크 3종 폴백 |
| F3 | 리스트 뷰 | 포스터 카드 그리드. 공연명·기간·장소·지역·가격·상태 배지·D-day·예매 버튼 |
| F4 | 월별 캘린더 뷰 | 월 단위 그리드에 공연 기간을 가로 바로 표시. 바 클릭 → 해당 카드로 스크롤/하이라이트 |
| F5 | 필터/정렬 | 상태 필터(예매중·공연예정·완료 숨김 토글), 지역 필터(시도), 정렬(공연일순/최근등록순) |
| F6 | 일일 자동 갱신 | GitHub Actions 10:00 KST. 마지막 갱신 시각을 페이지에 표시 |

### 제외 (Phase 2 이후)

- 티켓오픈일 수집·표시·알림 (예매처 공지 게시판 수집 필요 → 별도 설계)
- 텔레그램 신규 페스티벌 알림
- 사용자 즐겨찾기·계정 기능 일체

## 4. 데이터 소스 (확정)

KOPIS 오픈API. 인증키 발급 완료, `relates` 필드에 예매처 직링크 포함 확인됨
(검증 사례: PF294968 부산국제록페스티벌 → 예스24 `https://ticket.yes24.com/Perf/58933`).

주의: relates 는 공연마다 0~N개. 0개인 공연은 검색 딥링크 폴백으로 처리하며 UI에서
"예매처 검색" 이라고 구분 표기한다 (직링크인 척하지 않는다).

## 5. 페스티벌 판별 규칙

1. **축제목록 API** (`prffest`, `shcate=CCCD` 대중음악) 결과는 전량 채택
2. **공연목록 API** (`pblprfr`, `shcate=CCCD`) 결과 중 공연명이 포함 키워드에 걸리면 추가 채택
   - 포함: 페스티벌, FESTIVAL, FEST, 페스타, FESTA, 뮤직캠프, 록페, ROCK FES
   - 제외: 영화제, 필름, 불꽃, 맥주, 먹거리, 뮤지컬
3. mt20id 기준 중복 제거. 키워드는 `collector/config.py` 상수로 두어 코드 수정 없이 조정 가능

## 6. 성공 기준

- 펜타포트·부산록페·자라섬재즈 등 대표 페스티벌이 누락 없이 표시된다
- 카드 → 예매처 페이지 이동이 2클릭 이내
- 월 1회 미만의 수동 개입으로 운영 가능 (수집 실패 시 Actions 실패 메일로 인지)

## 7. 비목표 (명시적으로 안 함)

- 예매처 사이트 크롤링/스크래핑 (Phase 1에서는 어떤 형태로도 하지 않음)
- 티켓 가격 비교, 좌석 정보, 실시간 매진 여부
- 해외 페스티벌
